# Correctness Matrix

This matrix links correctness claims to the current validation stack. It is not a guarantee that every terminal dialect or host workflow is modeled; it is a review map for the package-owned terminal text kernel.

| Correctness area | Evidence | What is covered | Residual risk |
| --- | --- | --- | --- |
| Public core pipeline | `src/terminal-core.test.ts`, `tests/tui/public-layout.test.ts`, `bun run tui-oracle-check` | Prepare/layout/range/materialize agreement, hard breaks, soft hyphen behavior, tabs, start columns, source text reconstruction, and representative oracle comparison. | Oracle and fixtures cover representative terminal text cases, not every possible corpus. |
| Unicode width and grapheme safety | `src/terminal-string-width.test.ts`, `tests/tui/public-layout.test.ts`, `tests/tui/coordinate-projection.test.ts`, `bun run tui-fuzz --seed=ci --cases=2000` | CJK, emoji, combining marks, regional indicators, keycap clusters, zero-width marks, ambiguous-width policy, and grapheme-safe offset projection. | Width policy is explicit and configurable; consumers must pick a policy that matches their display expectations. |
| Source offset semantics | `tests/tui/coordinate-projection.test.ts`, `tests/tui/prepared-reader-boundary.test.ts`, `tests/tui/selection-extraction.test.ts` | UTF-16 source offsets, row/column projection, EOF projection, consumed wrap delimiters, source-range fragments, and extraction reconstruction. | Offsets are over sanitized visible source text; callers must keep external records aligned to that text. |
| Opaque handle boundaries | `tests/tui/public-api-boundary.test.ts`, `tests/tui/prepared-reader-boundary.test.ts`, `bun run api-snapshot-check`, `bun run package-smoke-test` | Public handles reject forged/cloned values where capability checks apply, internal prepared-reader state stays private, and package consumers use public entry points. | Incubating public type names may still be revised before a stable contract. |
| Large-text row lookup | `tests/tui/virtual-text.test.ts`, `tests/tui/layout-bundle.test.ts`, `tests/tui/performance-counters.test.ts` | Fixed-column sparse row indexes, page cache behavior, layout bundle page/projection agreement, invalidation generation checks, and page metadata. | Evidence is fixed-column by design; changing columns requires rebuilding width-dependent indexes. |
| Append-only flow parity | `tests/tui/chunked-append-parity.test.ts`, `tests/tui/layout-bundle.test.ts`, `bun run benchmark-check:tui`, `bun run memory-budget-check:tui` | Append-only flow agrees with full prepare at checkpoints, preserves stable prefix pages where valid, and rejects stale invalidation metadata. | Editing before the end of the source and destructive prefix eviction are not implemented. |
| Generic range index | `tests/tui/range-index.test.ts`, `tests/tui/search-session.test.ts`, `tests/tui/selection-extraction.test.ts` | Point/range overlap lookup, stable ordering, frozen cloned data, scoped search, and extraction range matches. | Range meaning is intentionally generic; callers own domain schema validation. |
| Source-first search | `tests/tui/search-session.test.ts`, `tests/tui/benchmark-config.test.ts`, `tests/tui/memory-budget.test.ts` | Literal/regex modes, case handling, whole-word filtering, scoped source ranges, projection metadata, navigation lookup, and modeled storage budget checks. | Current sessions store initial matches; bounded or streaming search design remains future stabilization work. |
| Selection/extraction | `tests/tui/selection-extraction.test.ts`, `tests/tui/coordinate-projection.test.ts`, `tests/tui/memory-budget.test.ts` | Coordinate-created selections, collapsed/forward/backward data, visible/source extraction, row fragments, and optional generic range matches. | Interaction state and copy formatting are host-owned and are not validated here. |
| Rich inline safety and provenance data | `src/terminal-rich-inline.test.ts`, `tests/tui/rich-security-gate.test.ts`, `tests/tui/selection-extraction.test.ts` | Profile validation, unsupported-control sanitize/reject modes, redacted diagnostics, URI policy, bidi format-control handling, limits, raw summaries, and explicit reconstruction options. | This is metadata extraction and policy enforcement, not a full terminal-emulation security model. |
| Evidence guardrails | `tests/tui/benchmark-evidence.test.ts`, `tests/tui/benchmark-claim-guard.test.ts`, `docs/evidence/benchmark-claims.md` | JSON report schema validation, provenance requirements, semantic matrix checks, and Markdown guards against copied timing values or overclaims. | Public prose still needs human review when new docs are added. |
| Release gate composition | `tests/tui/release-gate-consistency.test.ts`, `package.json` `release-gate:tui`, `bun run tui-corpus-check` | Confirms the package release gate includes typechecks, static gate, tests, oracle, corpus, fuzz, benchmark gate, memory-budget gate, demo check, API snapshot, and package smoke. | Optional local comparison reports remain outside the release gate. |
